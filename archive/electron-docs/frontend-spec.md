## 1. Overview

### 1.1 Purpose

This document defines the front-end user experience for Patient Explorer, a HIPAA-compliant desktop application for capturing, organizing, and searching patient data from multiple EMR sources.

### 1.2 Design Principles

| Principle | Description | Implementation |
|-----------|-------------|----------------|
| **Clinical Efficiency First** | Minimize clicks for common workflows | Paste â†’ Tag â†’ Save in 3 steps max |
| **Keyboard-First** | Power users expect shortcuts | All primary actions have hotkeys |
| **Offline-Aware** | Clear sync status visibility | Persistent status badge in header |
| **HIPAA-Safe Notifications** | No PHI in system alerts | Generic messages like "A colleague updated a chart" |
| **Error Prevention** | Guide users to success | Inline validation, confirmations for destructive actions |

### 1.3 Technology Stack

- **Framework**: React 18 + TypeScript 5.3+
- **Styling**: Tailwind CSS 3.x
- **Components**: Radix UI primitives + custom components
- **Icons**: Lucide React
- **State**: Zustand
- **Forms**: React Hook Form + Zod validation

## 6. Interaction Patterns

### 6.1 Keyboard Shortcuts

| Shortcut | Action | Context |
|----------|--------|---------|
| `Ctrl+V` | Paste from clipboard | Global |
| `Ctrl+S` | Save current form | Forms |
| `Ctrl+Shift+S` | Save and start new | Capture |
| `Ctrl+F` | Focus search | Global |
| `Ctrl+B` | Toggle batch mode | Capture |
| `Esc` | Close dialog/cancel | Dialogs |
| `Tab` | Next field | Forms |
| `Shift+Tab` | Previous field | Forms |
| `Enter` | Confirm/select | Dropdowns |
| `â†‘/â†“` | Navigate list | Dropdowns/lists |

### 6.2 Capture Flow

```
1. User presses Ctrl+V or drags image
2. Image preview appears instantly
3. OCR processing starts (show spinner)
4. If patient detected:
   - Auto-populate patient field
   - Show "OCR Match" banner with confidence
5. User confirms or changes patient
6. User optionally sets source/type
7. Ctrl+S to save
8. Toast confirms, auto-advance to next
```

### 6.3 Search Behavior

- **Debounce**: 150ms after typing stops
- **Fuzzy matching**: Tolerate typos, partial matches
- **Result order**: Best match first, then by last activity
- **Keyboard nav**: Arrow keys, Enter to select
- **Empty state**: Show "No patients found" + "Create new"

### 6.4 Sync Status Updates

- **Polling intervals**:
  - Focused + patient open: 2 seconds
  - Focused + no patient: 10 seconds
  - Background: 60 seconds
- **Status badge**: Always visible in header
- **Conflict notification**: PHI-safe toast
- **Checkout alerts**: Real-time via _changes feed
