# Workspace Reference

This folder is for **manually added** documents and reference artifacts that should be preserved without being affected by automated file management.

## Subfolders

| Folder | Purpose |
|--------|---------|
| `Research_Notes/` | Your research notes, findings, and analysis documents |
| `Templates/` | Reusable templates for documents, code, or workflows |
| `External_Docs/` | Documentation from external sources (APIs, libraries, vendors) |
| `Screenshots/` | Screenshots, diagrams, and visual references |
| `Code_Snippets/` | Useful code snippets for reference (not active project code) |
| `Meeting_Notes/` | Notes from meetings, calls, and discussions |
| `Vendor_Materials/` | Materials from vendors, partners, or third parties |

## Usage Guidelines

1. **Add files freely** - This folder is not managed by automated processes
2. **Organize by subfolder** - Use the appropriate subfolder for your content type
3. **Create new subfolders** - Add new subfolders as needed for your workflow
4. **Reference in sessions** - Point agents to specific files here when relevant

## Note

Files in this folder will NOT be:
- Automatically archived or moved
- Generated or overwritten by agents
- Included in automated cleanup processes

This is your persistent reference library.
